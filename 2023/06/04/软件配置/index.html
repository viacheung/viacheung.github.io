DEEPL

问题描述：使用DeepL翻译出来的文档是只读模式，显示不订阅pro版本之前都是不能编辑。

解决方法：将文档另存为.xml文档，然后右键用记事本打开，CTRL+F查找：w:documentProtection,，找到之后删除这一段完整标签<，保存，之后修改文档后缀名为.doc，打开之后发现文档已经可以编辑了。

navicat怎么导入sql数据库文件

  1. 打开Navicat，并连接数据库
  2. 鼠标右键“localhost”，点击“新建数据库”。
  3. 输入数据库名称，字符集选择“utf8”，排序规则选择“ utf8 general ci”，点击“确定”。
  4. 双击新建好的“数据库”。右键点击“运行SQL文件”。
  5. 导入SQL文件，点击开始。
  6. 右击表点刷新。

echo "# dqwd" >> README.md  #创建readme文件

#直接复制新建仓库的地址

把新文件上传到已有仓库

//master分支上
git init
git remote add origin git@github.com:ResetDing/test.git#直接github里面code下面ssh复制地址
git add .
git commit -m "first commit"
//获取github的分支
git fetch
//切换到远程你想切换的那个分支:（比如我想切换的是main分支为例 其他的你可以origin/XXX）
git checkout remotes/origin/main
现在可以看到自己的分支是一串数字字母
这时新建并切换到你想看的那个分支
git checkout main

git merge master --allow-unrelated-histories
//推送
git push -u origin main
